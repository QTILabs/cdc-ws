Backpressure & Resilience Mechanics

The architecture manages failures without complex distributed coordinators. The bounded Multi-Producer Single-Consumer (MPSC) channel dynamically translates downstream OpenSearch saturations into upstream database query halts.

MPSC Queue Depth Simulation

Nominal Flow Target Saturated Upstream Blocked
1

Target Saturation (HTTP 429)

OpenSearch cluster experiences high indexing load and rejects incoming HTTPS bulk requests.

2

Consumer Task Pauses

The daemon's consumer loop catches the exception, pauses, and initiates an exponential backoff sleep (2s to 60s).

3

MPSC Channel Accumulation

Because the consumer is asleep, the internal memory channel rapidly fills to its hard limit of 64 stream batches.

4

Producers Suspended

The Tokio runtime forces tx.send().await to block. Database FETCH queries halt. RisingWave safely buffers changelogs in its LSM storage.

⚠ Dead-Letter Queue (DLQ) Safeguard

If a single transaction fails due to dynamic index schema constraint rejections (poison pill), stalling the entire pipeline is unacceptable. The daemon intercepts item-level errors within the OpenSearch bulk HTTP 200 response.

➞

handle_dlq() extracts the original record, appends a UUID and timestamp, and routes it to an isolated -dlq index for secure administrative auditing via the BFF gateway.

Telemetry & OIDC Security

The platform enforces a strict Backend-for-Frontend (BFF) boundary for human interaction, while instrumenting all automated internal processes with standard OpenTelemetry (OTLP) metrics natively within the async Tokio runtime.

OTel Exported Metrics

  • pipeline_records_ingested Counter

    Total transaction records pulled from RisingWave cursors. Labeled by subscription.

  • pipeline_records_sunk_success Counter

    Transactions successfully mapped and indexed. Measures end-to-end processing rates.

  • pipeline_records_dlq Counter

    Poison pill documents safely isolated. Primary alert indicator for structural audits.

  • pipeline_mpsc_available_capacity Gauge

    Registered ONCE outside the loop to prevent OOM memory leaks via observer callbacks.
